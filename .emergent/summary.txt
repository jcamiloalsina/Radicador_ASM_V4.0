**Key Technical Concepts**
-   **API Pagination:** Serving large datasets in smaller, manageable chunks (pages) to improve initial load time and reduce server/client memory pressure.
-   **Progressive Data Loading:** A frontend strategy where data is fetched and rendered incrementally as it arrives from a paginated API, providing a fast initial UI response.
-   **In-Memory UI Caching:** Pre-calculating derived data (e.g., filtered lists) on the client-side to make UI interactions like filtering instantaneous (O(1) complexity).
-   **Offline-First:** Using IndexedDB (Dexie.js) and  to ensure application functionality and data persistence without a network connection.
-   **Stale-While-Revalidate (SWR):** Providing an instant UI response from cached data while fetching fresh data in the background.
