The user's primary goal is to build a comprehensive cadastral management web application, Asomunicipios, with a focus on the Conservación (Conservation) module. The main task is to implement a complete workflow for generating, verifying, and managing a Certificado Catastral Sencillo (Simple Cadastral Certificate).

**PRODUCT REQUIREMENTS (LATEST):**
1.  **PDF Generation:** Generate a Certificado Catastral Sencillo PDF. The final visual style (Style B - Institutional) has been chosen, and it replaces the director's signature with a digital verification block.
2.  **Digital Verification System:** The PDF must have a verifiable QR code and unique ID. The backend system for generating, publicly verifying, and annulling certificates must be fully functional.
3.  **Petition-Linked Workflow:**
    *   Users must be able to request a certificate via a Petición (Request), identifying the property by Código Predial Nacional (CPN) or Matrícula Inmobiliaria.
    *   The system must correctly find the property by searching for the Matrícula in the nested R1/R2 records.
    *   Staff must see a Generate Certificate button within the petition detail view.
    *   Generating a certificate from a petition must pre-fill the Radicado number on the PDF.
    *   Manual generation from the property viewer (with editable fields) must still be possible.
4.  **Workflow Automation:**
    *   Before staff generates a certificate, a confirmation modal must appear to verify that required documents and payment have been checked.
    *   Upon successful generation, the system must send the PDF certificate via email to the petitioner.
    *   The generated PDF must be attached to the petition, allowing the user to download it directly from the platform.
    *   The petition's status must be automatically set to Finalizado.
5.  **User Roles & Access:** A simplified view and workflow must exist for external users ( role), who can only create and view their own petitions.

**User's preferred language**: Spanish
